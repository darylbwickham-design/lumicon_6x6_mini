## Versioning notes

* **ESP firmware:** the **filename is the correct public version**. Internal header version strings are ignored where they conflict.
* **Plugin:** `lumi_con_bridge_integrated_v5_2` should be treated as **plugin version `1.0.0`**.

# Lumia Plugin Changelog

### 1.0.0

* Initial stable integrated Lumia plugin baseline.
* Added listener support for Lumi-Con device events.
* Added short and long press handling for the 6×6 matrix.
* Added device status variables for overlays, alerts, and debugging.
* Added TFT display actions for message, status, and clear operations.
* Added sequence dedupe handling to avoid repeated event processing.
* Fixed sequence dedupe after device reboot so events resume correctly after the ESP restarts.
* Included status, offline, and toast support in the integrated plugin branch.

### 1.1.0

* Renamed and repackaged the plugin as **6x6 matrix mini**.
* Expanded the plugin from matrix-only behavior into a fuller device plugin covering LCD, chat, alerts, and virtual pet support.
* Added pet event support with event range beginning at `72`.
* Added pet variables for overlays, alerts, and automation use.
* Added **Pet Action** support for status, sync, feed, play, clean, sleep, medicine, discipline, and reset.
* Added `petPath` advanced setting for ESP pet endpoint configuration.
* Updated debug toast wording to include pet-related activity.
* Expanded tutorials/settings references to external markdown files inside the plugin package.

### 1.1.1

* Refined the plugin manifest and alert definitions.
* Simplified pet alert variation metadata.
* Kept runtime behavior aligned with `1.1.0`.
* No core listener or action logic changes detected in `main.js`.

### 1.1.2

* Improved sequence handling for incoming device events.
* Changed duplicate detection so exact repeated sequence values are ignored cleanly.
* Reset sequence tracking when a lower sequence value is received, improving recovery after device-side counter changes.
* Greatly expanded the bundled setup and actions documentation.
* Added fuller documentation for key events, pet variables, device variables, and example Lumia setups.

### 1.1.3

* Added silent health-check support using the device status endpoint.
* Changed the idle/offline flow so the plugin performs a silent `/status?silent=1` check before marking the device offline.
* Added in-flight protection for the offline probe.
* Added helpers for base URL, timeout, and status-path handling.
* Updated unload behavior to silently clear device-connected state.
* Updated settings help text to document silent health checks and their dependency on the ESP base URL.
* Updated action and settings tutorials to explain the new health-check behavior.
